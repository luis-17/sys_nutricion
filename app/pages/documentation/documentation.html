Minotaur - Angular Admin Dashboard

Minotaur is premium admin dashboard powered with AngularJS. It's builded on popular Twitter Bootstrap v3 framework. Minotaur is fully based on HTML5 + CSS3 standards. Is fully responsive and clean on every device and every browser. It contains many example pages with many ready to use elements and is strongly customizable. You can choose choose from few premaid layouts such as horizontal navbar, boxed layout or rtl layout. Minotaur work on every modern browser, any device and any screen size.

Routing:

There are two important things about routing which are specific.

1. Defining specialClass parameter

specialClass parameter is used to place some route specific class to main html elements "html", "body", "minotaur-header" and "minotaur-nav". For example you can add parameter "specialClass: 'boxed-layout'" to your route so it will be loaded in boxed layout, this parameter "specialClass: 'force-header-sm header-sm'" load routed page with small header as default. Use specialClass "core" in case you want to load some kind of core page like login/signup so header/navigation won't be displayes in that case.

//layout-boxed
.state('layouts.boxed', {
  url: '/boxed',
  templateUrl: 'app/pages/layout-boxed/layout-boxed.html',
  controller: 'LayoutBoxedController',
  controllerAs: 'layout',
  parent: 'layouts',
  specialClass: 'boxed-layout'
})

//layout-header-sm
.state('layouts.header-sm', {
  url: '/header-sm',
  templateUrl: 'app/pages/layout-header-sm/layout-header-sm.html',
  controller: 'LayoutHeaderSmController',
  controllerAs: 'layout',
  parent: 'layouts',
  specialClass: 'force-header-sm header-sm'
})

//register
.state('pages.signup', {
  url: '/signup',
  templateUrl: 'app/pages/pages-signup/pages-signup.html',
  controller: 'SignupController',
  controllerAs: 'ctrl',
  parent: 'pages',
  specialClass: 'core'
})
2. Defining parent parameter

Parent parameter is used to keep dropdown menu opened while you are moving on a submenu level of same parent, for example if you are on common form elements page and you are moving on validation elements page which are both under same "forms" parent route. So don't forget to define it.

//form stuff
.state('forms', {
  url: '/app/forms',
  template: '
'
})
//common-elements
.state('forms.common', {
  url: '/common',
  templateUrl: 'app/pages/forms-common/forms-common.html',
  controller: 'FormsCommonController',
  controllerAs: 'forms',
  parent: 'forms'
})
//validation-elements
.state('forms.validation', {
  url: '/validation',
  templateUrl: 'app/pages/forms-validation/forms-validation.html',
  controller: 'FormsValidationController',
  controllerAs: 'forms',
  parent: 'forms'
})

Running Application

Run properly this application using Bower dependencies management tool and Gulp automation tool. Application is builded by Yeoman generator generator-gulp-angular

Required tools

After downloading source ZIP file from @Themeforest, you have to unzip it somewhere. We will be using non-builded application folder "admin/app".

  • As first make sure you have installed Node.js with package manager npm. You can check it in your terminal window using these commands node --version and npm --version. You can download Node.js here
  • As next step you need to install bower and gulp npm install -g bower gulp
  • After you have all tools installed, open your terminal window an go to application folder of our project "admin/app" as default, there have to be bower.json, package.json and gulpfile.js files located. There you need run this commands to install all dependencies npm install and bower install
  • Now you will be able to run your local server with this command gulp serve
Node.js

You can download Node.js here. Install it and check it in your terminal window with node --version and npm --version. If you get in some troubles try google it.

Now when you have node installed get to the application folder of our project "admin/app" as default in terminal window and run command npm install, there have to be package.json files located. Node will download and install all dependencies for you.

Bower

Bower works by fetching and installing packages from all over, taking care of hunting, finding, downloading, and saving the stuff you’re looking for. Bower keeps track of these packages in a manifest file, bower.json.

You can easily install Bower with npm running this command in terminal npm install -g bower

Now when you have bower installed get to the application folder of our project "admin/app" as default in terminal window and run command bower install, there have to be bower.json files located. Bower will download and install all dependencies from bower.json file for you.

If you want add and install any additional dependency you can do it by running this command in application folder bower install 'package name' --save

Gulp

Gulp is a JavaScript task runner (Automation), save yourself from repetitive tasks.

There are two important files for gulp gulpfile.js and package.json. gulpfile.js contains configurations for all plugins and tasks that gulp use. In package.json you can find all node packages that gulp needs.

As we have all configured in this application we can run localhost server with this command from application folder gulp serve

To build your application use this command from application folder gulp build

After bulding you'll get fresh copy of your builded application in "dist" folder located where your gulpfile.js file is.

Gulp build without minify/uglify

If you want build application with Gulp and don't want to minify/uglify files, you have to use command gulp build-nominify for compilation.

Layouts

Boxed Layout, add boxed-layout class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

Expanded Sidebar, add force-expanded nav-expanded class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

Small Header, add force-header-sm header-sm class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

Header Aside, add header-aside class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

Sidebar Aside, add sidebar-aside class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

Horizontal Menu, add hz-menu class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

RTL Layouts, add rtl class to html, body, minotaur-header and minotaur-nav elements or add it as specialClass parameter to route.

Pull content to header, add pull-to-header class to ".page" div.
